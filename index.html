EUD-6102 · rem-based banner font-size

Cookiebot Banner — Font-Size Test

Verifies the banner stays at 15px base font-size regardless of the host page's <html> font-size (Shopify-theme scenario).

1 · Host page <html> font-size

rem resolves against the root <html> element only. Pick a value, the banner is re-opened so displaydialog() re-measures.

Regression control — <body> font-size (must NOT affect banner)

Changing <body> font-size is the mistake to avoid — it changes em, not rem, so it never exercises the fix.

4 · Regression checklist

  1. All <html> values above resolve the dialog to 15px.
  2. Default page (100%) — nothing changed vs. previous behaviour.
  3. <body> override does NOT change the banner size.
  4. All banner templates: Swift, Elemental, US "Do not sell".
  5. Custom banners — inline font-size pin does not break intended sizing.
  6. Re-open dialog several times — no flicker / size reset.
  7. Widget and cookie declaration unaffected.
  8. Old cached banner.css + new cc.js still 15px.
  9. Inner elements (buttons, headings, tabs) scale proportionally.
  10. Browser minimum-font-size setting / page zoom.
